# Project Submission · Ship for Good 2026

## Deadline

**Saturday, May 30th · 19:00**

All work must be committed and pushed to your team branch before that time.

## Required README

Your branch README must include at minimum:

- Project and team name
- Problem it solves (one or two sentences)
- Prerequisites (language, runtime, dependencies)
- Installation and setup instructions
- Required environment variables (names only, no sensitive values)
- Main technologies used

**Extra credit (not required):**
- System architecture or diagram
- Key technical decisions
- Roadmap or next steps for anyone who wants to continue the project

## Live Demo

- The project must be **working**: deployed to production or running locally,
  but actually executing.
- Mockups, wireframes, non-functional prototypes, or theory-only projects
  are not valid submissions.
- If something is not finished, focus the demo on what does work.

**Showcase format:**
- 3 minutes of demo per team
- 3 minutes of jury feedback
- Show the main user flow — full coverage is not expected
