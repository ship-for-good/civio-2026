# Cursor Tokens

## How to apply the code
Info: https://cursorai.notion.site/Redeeming-Cursor-credits-32fda74ef045800dafe9ca9436bd8f8b

- The codes will be distributed during check-in on Friday, May 29.
- To redeem the credits, the user must be logged in in the same browser where they will open the credits URL. The same applies when using a QR code.
- As long as the credits have not been redeemed via the button, the user can open the same URL again in the same browser or in another one. It is important that they are logged in with the correct account, meaning a single-user account and not a team account.
- Credits can be redeemed on free or paid accounts and will appear in `Dashboard > Credits`. Redeeming them adds the credits to the user account, but does not apply them yet.
- If the attendee is on a free plan, they can use the free plan first and, once they reach the limit, they will need to start a Pro trial with a valid payment method for verification.
- The credits will be automatically applied to the next invoice.
  - If a free user starts a Pro trial or upgrades to a Pro account, the credits will be applied to the invoice once the subscription starts.
  - If the user already has a paid account, they will see the credits applied to the next invoice.
- If you do not see the credits in `Dashboard > Credits` after redeeming them, try a hard refresh or log out and log back into the Dashboard.
  - If that still does not work, try checking out on Stripe from your dashboard and see whether the credits are applied correctly there.
- The credits will not work if the user is on a Team plan; they are only valid for individual accounts such as Pro, Pro+, or Ultra.
- Note: certain features, such as `Background Agents` and `Web`, do not work without a paid plan.
