# Contributing · Ship for Good 2026

## Your branch

Each team has been assigned a dedicated branch: `team-XX-team-name`

**Work only on your own branch.** Do not commit to `main` or to
other teams' branches.

## Getting started

```bash
git clone https://github.com/shipforgood/hackathon-2025
cd hackathon-2025
git checkout team-XX-team-name
```

## Commit messages

Keep messages clear and descriptive. We recommend following the
[Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)
specification, but it is not required.

**Recommended format:**
| Type | When to use |
|------|-------------|
| `feat` | New functionality |
| `fix` | Bug fix |
| `docs` | Documentation changes |
| `refactor` | Code restructuring without behavior change |
| `chore` | Config, dependencies, scripts |

Examples:
- `feat: BOE procurement scraper`
- `fix: correct date parsing for Spanish format`
- `docs: add installation instructions`

What matters most: **anyone reading the history should understand
what changed and why.**

## Pushing your work

Push as often as you need throughout the hackathon:

```bash
git push origin team-XX-team-name
```

`main` is protected — direct pushes are not allowed. Do not open
Pull Requests during the event, they are not needed.
